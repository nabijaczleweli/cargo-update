Submodule ignore values

These values represent settings for the submodule.$name.ignore configuration value which says how deeply to look at the working directory when getting the submodule status.

Variants

Unspecified

Use the submodule's configuration

None

Any change or untracked file is considered dirty

Untracked

Only dirty if tracked files have changed

Dirty

Only dirty if HEAD has moved

All

Never dirty
